# 세움비즈 인증/권한 구조

이 문서는 세움비즈 업체 회원 시스템을 구현하기 전 인증, 권한, 세션 방향을 정리한 설계 초안입니다.
현재 단계에서는 실제 로그인 기능, Supabase Auth 연결, API, 서버, 화면 로직을 구현하지 않습니다.

## 1. 회원 유형

### 업체 회원

세움비즈에 상품권을 등록하고 정산을 신청하는 업체 단위 사용자입니다.

- 자기 업체의 등록내역 조회
- 자기 업체 잔액 조회
- 자기 업체의 잔액 로그 조회
- 자기 업체의 정산 신청 및 정산 이력 조회

### 관리자

세움비즈 운영자가 사용하는 관리 권한입니다.

- 전체 업체 조회
- 업체 잔액 조정
- 상품권 등록 승인/반려 처리
- 정산 기록 관리
- 업체 상태 관리

## 2. 업체 회원 기본 정보

업체 회원은 업체 단위 정보와 로그인 계정 정보를 함께 갖습니다.

| 항목 | 설명 |
| --- | --- |
| 업체명 | 업체 또는 상호명 |
| 아이디 | 로그인에 사용할 고유 아이디 |
| 비밀번호 | 로그인 비밀번호 |
| 담당자명 | 업체 담당자 이름 |
| 연락처 | 담당자 휴대폰 또는 대표 연락처 |
| 상태 | active, suspended, pending |
| 생성일 | 계정 생성 일시 |

업체 상태 코드 초안:

| 코드 | 화면 표시 | 의미 |
| --- | --- | --- |
| active | 정상 | 로그인 및 이용 가능 |
| pending | 승인대기 | 관리자 승인 전 |
| suspended | 정지 | 관리자에 의해 이용 정지 |
| withdrawn | 탈퇴 | 탈퇴 또는 이용 종료 |

## 3. 로그인 방식

### 1차

아이디와 비밀번호를 사용합니다.

- 업체별 고유 아이디로 로그인
- 비밀번호는 평문 저장 금지
- 실패 횟수 제한과 계정 잠금 정책은 추후 검토

### 2차 추후 검토

휴대폰 인증 가능성을 검토합니다.

- 담당자 휴대폰 본인 확인
- 비밀번호 재설정 보조 수단
- 고액 정산 신청 시 추가 인증 수단

## 4. 권한 범위

### 업체

업체 회원은 자기 업체 데이터만 접근할 수 있어야 합니다.

- 자기 등록내역만 조회
- 자기 업체 잔액만 조회
- 자기 로그만 조회
- 자기 정산 신청 내역만 조회

### 관리자

관리자는 운영 목적의 전체 데이터 접근 권한을 갖습니다.

- 전체 업체 조회
- 업체 잔액 조정
- 승인/반려 처리
- 정산 기록 관리
- 필요 시 업체 계정 정지 또는 승인 처리

## 5. 보안 원칙

- 업체끼리 데이터 절대 공유 금지
- 업체별 데이터 분리
- 관리자 권한 분리
- 업체 잔액 수정 로그 반드시 기록
- 업체 잔액은 단일 잔액 컬럼보다 로그 합계를 우선 기준으로 사용
- 관리자 조정, 승인, 반려, 정산 처리는 감사 로그를 남기는 방향으로 설계
- 비밀번호는 직접 구현 시 반드시 안전한 해시 방식 사용

## 6. 세션 방향

현재는 구조 설계만 진행합니다.

추후 후보:

- Supabase Auth
- 자체 JWT
- 서버 세션

검토 기준:

- 업체별 데이터 격리 구현 용이성
- 관리자 권한 분리 방식
- 비밀번호 재설정 및 휴대폰 인증 확장성
- RLS 정책과의 연결 가능성
- 운영자가 계정을 정지하거나 승인할 수 있는 구조

## 7. 상태값 설계

업체 상태:

- 정상
- 승인대기
- 정지
- 탈퇴

상태 흐름 초안:

| 현재 상태 | 가능한 다음 상태 | 설명 |
| --- | --- | --- |
| 승인대기 | 정상, 정지 | 관리자 승인 또는 보류 |
| 정상 | 정지, 탈퇴 | 운영 정책에 따른 제한 또는 이용 종료 |
| 정지 | 정상, 탈퇴 | 정지 해제 또는 이용 종료 |
| 탈퇴 | 없음 | 종료 상태 |

## 8. 후속 검토 항목

- 업체 계정과 업체 정보 테이블을 분리할지 여부
- 관리자 계정 구조를 기존 관리자 시스템과 통합할지 여부
- Supabase Auth 사용 시 `auth.users`와 업체 테이블 매핑 방식
- JWT 사용 시 토큰 만료, 재발급, 강제 로그아웃 방식
- 업체 회원가입을 직접 받을지, 관리자 생성 방식으로 시작할지 여부
- 정산 신청 시 추가 인증 적용 여부
